# Phase 1.1: Expression Parser Implementation

## 작업 배경
공학용 계산기의 핵심 로직을 안정적으로 처리하기 위해, 사용자 입력을 수학적 수식으로 변환하고 유효성을 검사하는 `ExpressionParser`가 필요합니다. TDD를 통해 다양한 수식 패턴(사칙연산, 삼각함수, 로그, 상수 등)을 정확하게 처리하는 파서를 구현합니다.

## 작업 내용 (TDD Tasks)
- [ ] **1.1.1** 테스트 파일 및 기본 구조 설정 (`tests/unit/core/ExpressionParser.test.js`)
- [ ] **1.1.2** 단일 숫자 파싱 테스트 ("5" → "5")
- [ ] **1.1.3** 단일 숫자 파싱 구현
- [ ] **1.1.4** 여러 자리 숫자 파싱 테스트 ("123" → "123")
- [ ] **1.1.5** 여러 자리 숫자 파싱 구현
- [ ] **1.1.6** 소수점 숫자 파싱 테스트 ("3.14" → "3.14", ".5" → "0.5")
- [ ] **1.1.7** 소수점 숫자 파싱 구현
- [ ] **1.1.8** 음수 파싱 테스트 ("-5" → "-5")
- [ ] **1.1.9** 음수 파싱 구현
