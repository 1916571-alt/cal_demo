# Phase 1.3: State Manager Implementation

## 작업 배경
애플리케이션의 상태(현재 값, 계산 내역, 설정 등)를 중앙에서 관리하고 UI 컴포넌트들에 변경 사항을 전달하는 역할을 합니다. 불변성(Immutability)을 유지하며 상태를 업데이트하고, 브라우저 저장소와의 연동을 통해 상태를 보존합니다.

## 작업 내용 (TDD Tasks)
- [ ] **1.3.1** 테스트 파일 설정 및 localStorage 모킹
- [ ] **1.3.2** 기본 상태 및 커스텀 초기화 테스트 및 구현
- [ ] **1.3.3** 상태 업데이트(setState) 단일/다중 프로퍼티 테스트
- [ ] **1.3.4** 상태 불변성 검증 테스트
- [ ] **1.3.5** 구독 시스템(Subscribe/Unsubscribe) 테스트 및 구현 (Observer pattern)
- [ ] **1.3.6** localStorage 저장 및 로드 테스트 (설정값 보존)
- [ ] **1.3.7** 오염된 데이터 처리 및 에러 핸들링 (localStorage)
- [ ] **1.3.8** 계산 히스토리 관리 (추가, 제한, 초기화) 테스트
- [ ] **1.3.9** 계산기 리셋 기능 테스트 및 구현

## 인수 조건
- [ ] 상태는 직접 변경할 수 없으며 `setState`를 통해서만 업데이트됨
- [ ] 상태 변경 시 등록된 모든 구독자(callback)에게 알림이 전달됨
- [ ] 각도 모드나 테마와 같은 사용자 설정이 페이지 새로고침 후에도 유지됨
- [ ] 최근 계산 내역이 최대 50개까지 저장되고 관리됨
- [ ] 코어 로직 테스트 커버리지가 90% 이상을 달성함
